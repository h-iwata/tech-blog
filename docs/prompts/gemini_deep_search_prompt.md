# Gemini Deep Search プロンプトテンプレート

以下の指示に従って、提供された記事の骨格（Skeleton）を基に深い調査を行い、得られた情報をAgentic RAG（llm-wiki）の形式で保存してください。

## 目的
提供された記事の骨格に基づいて関連する深い知識や最新動向、具体例を調査し、Wikiとして整理すること。

## 入力: 記事の骨格
```markdown
[ここに `latest/` にある記事の骨格やアイディアを貼り付けてください]
```

## 指示内容

1. **深いリサーチの実行**:
   - 上記の記事の骨格に関連する「技術的な詳細」「最新の動向」「具体的なユースケース」「実装のメリット・デメリット」について、Web検索等を用いて深く調査してください。
   - 特に一次情報や公式ドキュメント、信頼できる技術ブログからの情報を優先して取得してください。

2. **llm-wikiへの保存手順**:
   - 調査過程で得られた重要な生データ（記事の内容、公式ドキュメントのテキストなど）は、`llm-wiki/raw/` にわかりやすいファイル名で保存してください。
   - 生データを基に情報を構造化し、`llm-wiki/wiki/` の以下のいずれかのディレクトリにWiki記事を作成または更新してください：
     - `concepts/`: 技術概念やアーキテクチャ
     - `entities/`: 特定のツール、ライブラリ、企業
     - `sources/`: 取得した特定の記事の要約
     - `synthesis/`: 複数の情報を横断した分析

3. **保存時のルール**:
   - 既存の `LLM_WIKI_GUIDELINES.md` に必ず従ってください。
   - Wikiページは日本語で記述し、`raw/` データの参照パス（例: `[参考](../../raw/xxx.md)`）を必ず記載してください。
   - 調査完了後、`npx qmd update` および `npx qmd embed` を実行し、インデックスを最新化してください（このリポジトリのルートで実行）。

4. **出力**:
   - 作成または更新したWikiページのパス一覧と、その簡単な要約を報告してください。
