# FastLabelブログ 執筆ガイドライン (Hatena Blog向け)

本ガイドラインは、FastLabelの公式テックブログ（[https://fastlabel.ai/blog](https://fastlabel.ai/blog)）に準拠した記事をHatena Blog向けに作成・投稿するための統一基準です。AIエージェントが記事を生成する際は、以下のルールに厳密に従ってください。

## 1. 記事の基本構成と分量
- **全体文字数**: 3,000文字 〜 5,000文字程度（B2Bの専門記事として十分な深さを持たせる）
- **目次**: 記事の冒頭（リード文の直後）にHatena Blogの目次記法 `[:contents]` を配置してください。
- **構成**:
  1. **タイトル**（30〜40文字、検索キーワードを含める）
  2. **リード文**（読者の課題提起と解決策の提示、300文字程度）
  3. **目次** (`[:contents]`)
  4. **前提・背景**（業界動向や技術の基礎）
  5. **本題・詳細解説**（技術アーキテクチャ、ユースケース等）
  6. **FastLabelのソリューション**（アノテーションやAIデータ基盤の紹介）
  7. **まとめ**（次のアクションへの誘導）

## 2. タグの種類と付与基準
Hatena Blogのカテゴリ/タグ機能として、以下のいずれか（複数可）を想定して記事にメタデータとして記載してください。
- `アノテーション` (Annotation)
- `生成AI` (Generative AI)
- `自動運転` (Autonomous Driving)
- `ロボティクス` (Robotics)
- `画像認識` (Computer Vision)
- `MLOps` / `データセット`

## 3. 画像の品質と分量
- **サムネイル画像**: 記事の冒頭に、高解像度（横1200px x 縦630px推奨）で、フラットデザインまたはアイソメトリックなイラストを用いた高品質な画像を1枚設定します。
- **記事内の画像量**: 見出し（H2）ごとに1枚程度、または複雑なアーキテクチャやUIを説明する箇所に必ず図解（概念図やフローチャート）を挿入してください。（目安: 1記事あたり3〜5枚）
- **画像の代替テキスト**: アクセシビリティのため、必ず `![代替テキスト](画像URL)` の形式で簡潔な説明を含めてください。

## 4. 文体 (Tone & Voice)
- **語り口**: 「です・ます」調。
- **トーン**: 専門的かつ論理的でありながら、初心者にも配慮した「信頼感のある技術アドバイザー」のトーン。主観的・感情的な表現は避け、客観的なデータや一次情報に基づいた記述を心がけてください。

## 5. リンクと参照の作法

### 外部記事・参考リンクへの参照 (Hatena Blog記法)
他のWebページや技術ドキュメントを参照する場合は、Hatena Blogのブログカード記法を活用して視覚的に分かりやすくします。
```markdown
[https://example.com:embed:cite]
```
または、シンプルなテキストリンクの場合は以下を使用します。
```markdown
[https://example.com:title]
```

### LLM Wiki（元データ）への参照
AIがナレッジベースから引き出した情報（`llm-wiki/raw/` のデータ）を利用した箇所には、記事の末尾の「参考文献」セクションに、RAGの証拠としてファイルパスを明記してください。これはファクトチェックのトレーサビリティを保つためです。
```markdown
## 参考文献・元データ
- [参考: 自動運転データセット仕様](../../llm-wiki/raw/autonomous-driving-specs.md)
```

## 6. AIエージェントへの出力指示まとめ
記事を最終出力する際は、そのままHatena Blogのエディタに貼り付けられる純粋なMarkdown形式（Hatena記法含む）で出力してください。Markdownのコードブロック（````markdown ... ````）で囲って出力すると、コピー＆ペーストが容易になります。
